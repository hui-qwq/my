# my

一个用于记录 **C++ 学习、底层实现练习与 Linux 网络编程实践** 的个人仓库。

这个仓库不是单一项目，而是我在不同阶段围绕 **C++ 基础能力、STL 实现原理、Linux 网络编程** 所做的练习集合。不同分支对应不同主题，既包含对底层原理的拆解，也包含从知识到项目落地的实践。

---

## 仓库说明

目前仓库主要包含两个核心练习方向：

- **`master`**：手写 STL / `MyTinySTL` 学习与实现练习
- **`mini-chat-server`**：基于 Linux 网络编程的简易聊天室项目

默认分支的 README 主要用于介绍整个仓库的学习路线与分支定位。

---

## 分支概览

### 1. `master` —— MyTinySTL / 手写 STL 练习

这个分支主要用于跟着 `MyTinySTL` 进行 STL 底层实现学习，重点理解：

- 泛型编程与模板基础
- 容器的内部数据结构
- 迭代器设计
- 对象构造与析构
- 内存分配与资源管理
- 容器插入、删除、扩容过程中的生命周期控制

#### 当前学习重点

目前阶段性完成和重点消化的内容包括：

- `vector`
- `list`
- `construct`
- `allocator`

这个分支更偏“**底层原理拆解 + 代码实现 + 单元测试验证**”，适合用来展示我对 C++ 基础与 STL 设计思路的理解。

---

### 2. `mini-chat-server` —— Linux 简易聊天室

这个分支是一个基于 Linux 网络编程实现的小型聊天室项目，目标是把网络编程里的核心知识点串起来，并通过实际代码完成一次完整练习。

#### 已实现功能

- 基于 TCP 的客户端 / 服务端通信
- 服务端支持多个客户端同时连接
- 使用 `epoll` 进行 I/O 多路复用
- 聊天消息广播
- 客户端断开后的资源清理
- 客户端同时处理终端输入和服务端消息
- 使用 `pipe + splice` 优化客户端输入转发

这个分支更偏“**Linux 系统编程 + 网络通信 + 小项目落地**”，重点体现对 socket、非阻塞 I/O、多路复用机制的理解和使用。
