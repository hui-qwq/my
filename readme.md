# mini-chat-server

用来练习 Linux 网络编程的小项目，一个最简单的聊天室。

项目一开始是用 `epoll` 写的，后来为了学习事件驱动模型，改成了基于 `libevent` 的版本。

---

## 现在能做什么

目前实现的是一个最基础的多人聊天室：

- 服务端可以同时处理多个客户端连接
- 客户端发送消息后，会广播给其他客户端
- 客户端断开后，服务端会清理资源
- 如果客户端长时间没有发消息，会被服务器踢掉（超时机制）

---

## 技术点

这个项目主要用来练这些东西：

- TCP 网络编程
- I/O 多路复用（最开始用 epoll）
- libevent 基本使用
- bufferevent 的读写模型
- 事件驱动编程（callback）
- 定时器事件（用来做超时检测）

---

## 服务端大概是怎么工作的

我现在是这样拆的：

- `accept_cb`：有新客户端连接进来
- `read_cb`：客户端发消息
- `event_cb`：客户端断开 / 出错
- `timer_cb`：定时检查哪些客户端超时

服务端内部维护了两个表：

- `clients`：当前在线的连接
- `lastActive`：每个客户端最后一次活跃时间

定时器每隔一段时间扫一遍，如果太久没发消息，就踢掉。
